# 投资论点跟踪

## 核心理念

首推报告里写的"3-5 个核心论点"不能放在那里就完事。每个论点要分解成**可观测、可验证、可触发警告**的具体指标。

## 论点分解模板

每个论点拆成 4 个要素：

```
论点：{一句话陈述}
依据：{当前的数据 / 趋势 / 政策}
验证指标：{可量化的领先 / 同步指标}
反证条件：{什么情况下论点失效}
```

### 示例
```
论点：公司将受益于新能源汽车渗透率提升
依据：2024 新能源车销量同比 +35%，预期 2025 渗透率达 50%
验证指标：
  - 月度新能源车销量数据
  - 公司新能源相关产品收入占比（季报披露）
  - 主要客户（如比亚迪 / 宁德）的订单情况
反证条件：
  - 新能源车销量增速 < 15%
  - 公司新能源业务毛利率连续两季度下滑 > 3pct
  - 主要客户转单 / 验证不通过
```

## 跟踪频率

| 论点类型 | 跟踪频率 |
|---|---|
| 季度财报指标 | 每季度 |
| 月度运营数据 | 每月 |
| 行业景气指标 | 每月 |
| 政策催化 | 实时 |
| 客户 / 订单 | 每月 / 每季 |

## 论点状态管理

| 状态 | 判定 |
|---|---|
| 强化 | 验证指标向好且加速 |
| 维持 | 验证指标符合预期 |
| 减弱 | 验证指标走弱但未触反证 |
| 失效 | 触反证条件 |

论点失效时强制要求：
1. 移除该论点
2. 重新评估整体覆盖逻辑
3. 决定是否调整覆盖评级（不在 agent 层面做，提交人工审议）

## 输出结构

```
## {标的} 论点跟踪 - {YYYY-MM-DD}

### 论点 1：{论点陈述}
- 当前状态：维持 / 强化 / 减弱 / 失效
- 验证指标最新值：
  - 指标 A：当前 X，符合 / 偏离预期
  - 指标 B：...
- 本期评估：
  - 进展：... 
  - 风险：...
- 下次跟踪：{时间} / {触发事件}

### 论点 2：{论点陈述}
（同结构）

### 整体覆盖逻辑评估
- 论点存活情况：N/M
- 是否需要重新评估覆盖
- 重大变化提示

### 跟踪行动
- [ ] 下季报后更新论点 1
- [ ] 关注 X 客户订单情况
- [ ] 关注 Y 政策落地节奏
```
