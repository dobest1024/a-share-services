# 客户画像

## 标准画像维度

### 1. 基本信息
- 年龄 / 性别
- 职业（是否影响投资能力 / 时间）
- 家庭结构（已婚 / 子女 / 父母赡养）
- 居住地（影响税务 / 资产配置）

### 2. 财务状况
- 当前可投资资产规模
- 年收入 / 月度结余
- 负债（房贷 / 车贷 / 信用卡）
- 已有资产配置（存款 / 房产 / 股票 / 基金 / 保险）

### 3. 投资目标
- 退休
- 子女教育
- 大额支出（购房 / 创业）
- 财富传承
- 资产保值
- 财富增长

### 4. 时间跨度
- 短期（< 3 年）：保本为主
- 中期（3-7 年）：均衡配置
- 长期（> 7 年）：可承受短期波动

### 5. 风险承受能力
监管要求的"投资者适当性"分级：
- C1 谨慎型（保本）
- C2 稳健型
- C3 平衡型
- C4 积极型
- C5 激进型

判断维度：
- 财务承受能力（年收入 / 净资产 / 持续收入能力）
- 投资经验
- 风险偏好（主观）
- 流动性需求

### 6. 流动性需求
- 紧急备用金需求
- 计划内大额支出时点
- 预留可周转的现金比例

### 7. 税务情形
- 个人所得税适用税率
- 是否有税收优惠（专项附加扣除）
- 资本利得相关考虑（A 股个人投资者免征）

## 适当性匹配（监管要求）

证监会《证券期货投资者适当性管理办法》要求：
1. 经营机构必须了解客户
2. 给客户匹配合适的产品（不能错配）
3. 留存匹配记录与签字
4. 风险揭示书必须签字

匹配示例：
- C1 谨慎型客户 → 仅可购买保本类产品
- C5 激进型客户 → 可购买高风险产品（含杠杆 / 衍生品）

## 工作流

1. 与客户交互问询（用问卷或结构化对话）
2. 根据回答打分得到画像
3. 套用监管分级
4. 输出"画像报告"，由持牌投顾签字确认

## 输出结构

```
## 客户画像报告 - {客户编号} - {YYYY-MM-DD}

### 基本信息
（脱敏后保存）

### 财务状况
- 可投资资产：X 万元
- 年收入：X 万元
- 资产负债率：X%

### 投资目标
- 主要目标：{...}
- 期望年化收益区间：X% ~ Y%
- 可接受最大回撤：X%

### 适当性等级
- 评估结果：C1 / C2 / C3 / C4 / C5
- 评估依据（量表分数）：
- 建议产品类型：{...}

### 流动性需求
- 紧急备用：X 个月支出
- 短期支出计划：{...}

### 后续建议
- 配置框架方向（用 allocation-framework skill 展开）

### ⚠️ 合规
- 本画像由 {持牌投顾} 评估

```
